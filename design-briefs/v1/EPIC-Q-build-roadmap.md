**Master spec reference:** Section Q.

## Purpose

Informational. The build roadmap is a sprint-planning instrument. Designers consult it when scoping a sprint's design deliverable to ensure visual work lands in the same sprint as the implementing dev work.

## How designers use this brief

- A Phase 1 sprint requires the marketing-site surfaces designed (portal, vertical landings, contact, locations, knowledge hub, legal).
- A Phase 2 sprint requires the property catalogue, property detail and area-guide detail designed.
- A Phase 3 sprint requires the admin shell, property editor, content editor designed.
- A Phase 4 sprint requires every public form's design plus the CRM queue and detail.
- A Phase 5 sprint requires the repair flow (tenant form, admin inbox, contractor portal) designed.
