**Master spec reference:** Section B.
**Pack:** Core.
**Status:** CONTEXT (no standalone visual surface).

## Purpose

Informational. The feature audit is the inventory designers should consult before opening any specific epic's design.

## How designers use this brief

- When picking up a design ticket, look up the feature in master spec Section B to understand: where it appears, what the user does, and what happens after submission. The functional context constrains the design.
- The Must / Should / Nice priority guides depth — a Must feature gets full state coverage (loading, empty, error, success), a Nice feature may defer empty-state designs to V2.
