# EPIC-J — Data requirements (design)

**Dev brief:** [dev-briefs/v1/EPIC-J-data-requirements.md](../../dev-briefs/v1/EPIC-J-data-requirements.md).
**Master spec reference:** Section J.
**Status:** CONTEXT (no standalone visual surface).

## Purpose

Informational. The data model itself has no visual surface but it constrains every form layout in the admin. Designers consult this brief to understand: how many fields exist on an entity, which fields are required and which are optional, which fields are conditionally relevant per `listing_type`, and which fields are computed.

## How designers use this brief

When designing an admin form, the field set is the field set declared in master spec Section J for that entity. Field grouping (and tab structure for large entities like Property) is a design decision; the field roster itself is not.
