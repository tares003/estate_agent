Transactional email templates

EPIC-D / EPIC-AC — brand-consistent transactional emails (Phase B)
Templates
enquiry acknowledgement · viewing confirmation · repair status update · feedback request
Build
table-based, inline styles, 600px body — email clients strip <link> & CSS variables, so token VALUES are mirrored inline
Responsive (RULE ZERO)
each shown at a ~375px mobile client and a ~600px desktop client side by side
Why these look different from the rest of the canvas. Email clients (Outlook, Gmail, Apple Mail) strip <link> stylesheets and don't support CSS custom properties, flexbox or grid reliably. So these templates are built the email-safe way: table layout, a 600px body, and inline styles. The values are the same tokens as the rest of the system (brand #1F2937, accent #C9A24B, surfaces, Fraunces/Inter web fonts with serif/sans fallbacks) — just inlined. Each template is shown at a phone-width and a desktop-width client per RULE ZERO.
