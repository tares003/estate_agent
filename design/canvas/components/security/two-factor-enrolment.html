The four-step flow

Single-column on mobile; steps transition over --motion-duration-base. Designer interpretation (open question 3): V1 ships TOTP only — a QR + manual key for any authenticator app. WebAuthn is a later addition; the layout leaves room for an "Or use a security key" alternative.

Step 1 of 4

Protect your account

Your role requires two-factor authentication. You'll enter a 6-digit code from an authenticator app each time you sign in. It takes about two minutes to set up.

Tokens: step card --radius-lg + --colour-border · step badge --colour-brand-accent · code input focus --colour-brand-accent + --shadow-focus, error --colour-danger · backup codes --colour-surface-sunken + --font-mono · step transition --motion-duration-base.
Accessibility: the QR is accompanied by a readable manual key (a11y rule: text equivalent). Code entry is a labelled input group; the "code incorrect" message uses role="alert". Backup codes can't be dismissed without the explicit "I've saved these" affirmation. Every control is keyboard-reachable.
